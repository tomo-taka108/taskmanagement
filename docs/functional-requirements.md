## 1. 機能要件

### 1.1 カラム（リスト）機能

アプリ起動時に「未着手 / 作業中 / 完了」の3列をデフォルトで表示する。カラムは自由に追加・削除・名前変更が可能。

| ID | 機能 | 優先度 | 実装状況 | 受入条件 |
|----|------|--------|----------|----------|
| C-01 | カラムを追加できる | 必須 | ✅ 実装済み | 名前を入力して追加するとボード右端に新しいカラムが表示される |
| C-02 | カラムを削除できる | 必須 | ✅ 実装済み | 削除後、カラムと配下のカードがすべて消える |
| C-03 | カラム名を変更できる | 必須 | ✅ 実装済み | カラム名をクリック/編集して確定すると表示が更新される |
| C-04 | カラムの順序を変更できる（ドラッグ&ドロップ） | 必須 | ✅ 実装済み | カラムをドラッグして別の位置にドロップすると順序が変わりDBに保持される |

### 1.2 カード（タスク）機能

| ID | 機能 | 優先度 | 実装状況 | 受入条件 |
|----|------|--------|----------|----------|
| K-01 | カードを作成できる | 必須 | ✅ 実装済み | タイトル（必須）・説明・優先度・期限日を入力して追加するとカラム末尾にカードが追加される |
| K-02 | カードを編集できる | 必須 | ✅ 実装済み | カードをクリックすると詳細モーダルが開き、タイトル等を編集・保存できる |
| K-03 | カードを削除できる | 必須 | ✅ 実装済み | カード詳細モーダルの削除ボタンをクリックすると確認ダイアログが表示され、確認後にカードが一覧から消えDBからも物理削除される |
| K-04 | カードをカラム間でドラッグ&ドロップで移動できる | 必須 | ✅ 実装済み | カードを別カラムにドロップすると移動し、DBのカラムIDが更新される |
| K-05 | カラム内でカードの順序を変更できる（ドラッグ&ドロップ） | 必須 | ✅ 実装済み | カード間にドロップすると指定した位置に挿入され順序がDBに保持される |

### 1.3 カード詳細機能

| ID | 機能 | 優先度 | 実装状況 | 受入条件 |
|----|------|--------|----------|----------|
| D-01 | カードに説明文を入力できる | 高 | ✅ 実装済み | 説明文を入力・保存するとモーダル再表示時に内容が保持されている |
| D-02 | カードに期限日（Due date）を設定できる | 高 | ✅ 実装済み | 日付を設定・保存するとカード一覧上に期限日が表示される |
| D-03 | 期限超過時に視覚的な警告を表示する | 中 | ✅ 実装済み | 期限日を過ぎたカードに赤色のバッジや警告アイコンが表示される |
| D-04 | カードにラベルを設定できる | 高 | 部分実装 | ラベルを選択・保存するとカード上にラベルが色付きで表示される（表示は実装済み。モーダルからの編集は未実装） |
| D-05 | カードにチェックリスト（サブタスク）を追加できる | 中 | 部分実装 | 項目を追加・チェックでき、完了状態がDBに保持される（表示は実装済み。モーダルからの追加・編集は未実装） |
| D-06 | チェックリストの完了率をカード上に表示する | 低 | ✅ 実装済み | 完了率がカード一覧上にプログレスバーで表示される |
| D-07 | カードに優先度を設定できる | 高 | ✅ 実装済み | 優先度（高・中・低）を設定・保存するとカード上にバッジで表示される |

### 1.4 ラベル機能

プリセットラベルを用意しつつ、ユーザーが独自のラベルを追加作成できる。ラベルはアプリ全体で共通管理する（ボード単位の管理は不要）。

| ID | 機能 | 優先度 | 実装状況 | 受入条件 |
|----|------|--------|----------|----------|
| L-01 | プリセットラベルを選択できる | 高 | 部分実装 | 定義済みの色・名前のラベルをカードに付与できる（表示は実装済み。付与操作は未実装） |
| L-02 | カスタムラベルを名前・色指定で作成できる | 中 | 未実装 | 名前と色を指定して作成するとラベル一覧に追加され、カードに付与できる |
| L-03 | カスタムラベルを編集・削除できる | 中 | 未実装 | 編集後に名前・色が反映され、削除後は該当カードからも除去される |

### 1.5 フィルタ・検索機能

| ID | 機能 | 優先度 | 実装状況 | 受入条件 |
|----|------|--------|----------|----------|
| F-01 | キーワードでカードを検索できる | 中 | ✅ 実装済み | 検索バーに入力するとタイトル・説明文にマッチするカードのみ表示される |
| F-02 | ラベルでカードをフィルタリングできる | 中 | ✅ 実装済み | ラベルを選択するとそのラベルを持つカードのみ表示される |
| F-03 | 期限日でカードをフィルタリングできる | 低 | ✅ 実装済み | 「期限超過」「今週中」等の条件でカードを絞り込める |

### 1.6 UI/UX機能

| ID | 機能 | 優先度 | 実装状況 | 受入条件 |
|----|------|--------|----------|----------|
| U-01 | ダークモード / ライトモードを切り替えられる | 中 | ✅ 実装済み | 切り替えボタンで即座にテーマが変わり、設定がlocalStorageに保持される |
| U-02 | カードの色を設定できる | 低 | 未実装 | カードの背景色を選択すると一覧上で反映される |
| U-03 | レスポンシブデザイン（PC・タブレット対応） | 中 | 未実装 | タブレット幅（768px以上）でレイアウトが崩れず使用できる |

## 2. 非機能要件

| 区分 | 要件 |
|------|------|
| パフォーマンス | カードが100枚以下であればストレスなく動作すること |
| データ永続化 | アプリを再起動してもDBのデータが保持されること |
| ブラウザ対応 | Chrome・Edge・Safariの最新版をサポート |
| アクセシビリティ | キーボード操作でカードの作成・削除が可能なこと |
| API | RESTful設計に従い、HTTPステータスコードを適切に返すこと |
