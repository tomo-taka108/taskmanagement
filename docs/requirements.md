# タスク管理アプリ 要件定義書

**バージョン:** 0.1  
**作成日:** 2026-04-29  

---

## 1. プロジェクト概要

### 1.1 目的
Trelloライクなカンバン形式のタスク管理アプリを構築する。
チームや個人がタスクをカード形式で管理し、進捗を視覚的に把握できるようにする。

### 1.2 スコープ
- MVP機能（ボード・カラム・カードの基本操作）
- 追加機能（カード詳細・フィルタ・検索・ダークモード）
- 複数ボード対応
- バックエンド・マルチユーザー対応は対象外

## 3. 機能要件

### 3.1 ボード機能

複数ボードに対応する。ボード一覧画面から各ボードへ遷移する構成とする。

| ID | 機能 | 優先度 |
|----|------|--------|
| B-01 | ボード一覧を表示できる | 必須 |
| B-02 | ボードを新規作成できる | 必須 |
| B-03 | ボードのタイトルを変更できる | 必須 |
| B-04 | ボードを削除できる | 必須 |

### 3.2 カラム（リスト）機能

初期状態として「Todo / In Progress / Done」の3列をデフォルトで用意する。カラムは自由に追加・削除・名前変更が可能。

| ID | 機能 | 優先度 |
|----|------|--------|
| C-01 | カラムを追加できる | 必須 |
| C-02 | カラムを削除できる | 必須 |
| C-03 | カラム名を変更できる | 必須 |
| C-04 | カラムの順序を変更できる（ドラッグ&ドロップ） | 必須 |
| C-05 | 新規ボード作成時にデフォルト3列（Todo / In Progress / Done）が自動生成される | 必須 |

### 3.3 カード（タスク）機能

| ID | 機能 | 優先度 |
|----|------|--------|
| K-01 | カードを作成できる | 必須 |
| K-02 | カードを編集できる | 必須 |
| K-03 | カードを削除できる | 必須 |
| K-04 | カードをカラム間でドラッグ&ドロップで移動できる | 必須 |
| K-05 | カラム内でのカードの順序を変更できる（ドラッグ&ドロップ） | 必須 |

### 3.4 カード詳細機能

| ID | 機能 | 優先度 |
|----|------|--------|
| D-01 | カードに説明文を入力できる | 高 |
| D-02 | カードに期限日（Due date）を設定できる | 高 |
| D-03 | 期限超過時に視覚的な警告を表示する | 中 |
| D-04 | カードにラベルを設定できる | 高 |
| D-05 | カードにチェックリスト（サブタスク）を追加できる | 中 |
| D-06 | チェックリストの完了率をカード上に表示する | 低 |

### 3.5 ラベル機能

プリセットラベルを用意しつつ、ユーザーが独自のラベルを追加作成できる。

| ID | 機能 | 優先度 |
|----|------|--------|
| L-01 | プリセットラベル（赤・黄・緑・青・紫など）を選択できる | 高 |
| L-02 | カスタムラベルを名前・色指定で作成できる | 中 |
| L-03 | カスタムラベルを編集・削除できる | 中 |
| L-04 | ラベルはボード単位で管理される | 必須 |

### 3.6 フィルタ・検索機能

| ID | 機能 | 優先度 |
|----|------|--------|
| F-01 | キーワードでカードを検索できる | 中 |
| F-02 | ラベルでカードをフィルタリングできる | 中 |
| F-03 | 期限日でカードをフィルタリングできる | 低 |

### 3.7 UI/UX機能

| ID | 機能 | 優先度 |
|----|------|--------|
| U-01 | ダークモード / ライトモードを切り替えられる | 中 |
| U-02 | カードの色を設定できる | 低 |
| U-03 | レスポンシブデザイン（PC・タブレット対応） | 中 |

---

## 4. 非機能要件

| 区分 | 要件 |
|------|------|
| パフォーマンス | カードが100枚以下であればストレスなく動作すること |
| データ永続化 | ブラウザを閉じてもデータが保持されること（localStorage） |
| ブラウザ対応 | Chrome・Edge・Safariの最新版をサポート |
| アクセシビリティ | キーボード操作でカードの作成・削除が可能なこと |

## 5. データモデル

```typescript
// アプリ全体の状態
interface AppState {
  boards: Record<string, Board>;
  columns: Record<string, Column>;
  cards: Record<string, Card>;
  labels: Record<string, Label>;  // ボードIDをキーにラベルを管理
  boardOrder: string[];           // ボード一覧の並び順
}

// ボード
interface Board {
  id: string;
  title: string;
  columnIds: string[];            // カラムの並び順
}

// カラム
interface Column {
  id: string;
  title: string;
  cardIds: string[];              // カードの並び順
}

// カード
interface Card {
  id: string;
  title: string;
  description?: string;           // 省略可能（説明文なしのカードも存在するため）
  dueDate?: string;               // 省略可能、ISO 8601形式
  labelIds: string[];
  checklists: ChecklistItem[];
  color?: string;                 // 省略可能、HEX形式
}

// ラベル
interface Label {
  id: string;
  boardId: string;                // どのボードに属するか
  name: string;
  color: string;                  // HEX形式
  isPreset: boolean;              // プリセットか否か
}

// チェックリスト項目
interface ChecklistItem {
  id: string;
  text: string;
  completed: boolean;
}
```

**プリセットラベル例:**

| 名前 | カラー |
|------|--------|
| 赤 | `#ef4444` |
| 黄 | `#eab308` |
| 緑 | `#22c55e` |
| 青 | `#3b82f6` |
| 紫 | `#a855f7` |
| グレー | `#6b7280` |
