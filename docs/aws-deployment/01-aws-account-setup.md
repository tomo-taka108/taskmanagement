# 01. AWSアカウント作成 〜 IAM Identity Center設定（手動作業）

このステップは**ブラウザでの手動作業**です。Claude Codeはこの作業を代行できません
（アカウント作成にはメールアドレス確認・クレジットカード登録・本人確認が必要なため）。
ここに書かれた手順に沿って、あなた自身が進めてください。

完了したら `02-cli-setup.md` に進み、そこからは再びコマンド操作に戻ります。

---

## 1. AWSアカウントを作成する

1. https://aws.amazon.com/jp/ にアクセスし、「無料アカウントを作成」をクリック
2. メールアドレス、パスワード、アカウント名を入力
3. 連絡先情報（住所等）を入力
4. **クレジットカード情報を登録**（本人確認のため必須。今回の構成なら学習中の課金は数十円〜数千円程度に収まる想定ですが、`07-cost-management.md`のコスト管理を必ず実践してください）
5. 電話番号によるSMS/音声認証で本人確認
6. サポートプランを選択（「Basic support - free」で問題ありません）

作成が完了すると、「ルートユーザー」としてログインできる状態になります。

## 2. ルートユーザーにMFA（多要素認証）を設定する

ルートユーザーはアカウントに対して最強の権限を持つため、乗っ取られると被害が甚大です。
必ずMFAを設定してください。

1. マネジメントコンソールにルートユーザーでログイン
2. 右上のアカウント名 → 「セキュリティ認証情報」
3. 「多要素認証 (MFA)」セクションで「MFAデバイスを割り当てる」
4. スマートフォンの認証アプリ（Google Authenticator、Authyなど）で仮想MFAデバイスを設定

以降、**日常的な作業ではルートユーザーを使いません**。次に作る「自分専用ユーザー」を使います。

## 3. IAM Identity Center を有効化する

IAM Identity Center（旧名: AWS SSO）は、AWSへのログインとアクセス権限をまとめて管理する仕組みです。
今回はこれを使って、AWS CLIからブラウザ認証でログインできるようにします。

1. マネジメントコンソールの検索バーで「IAM Identity Center」を検索して開く
2. 「有効にする」をクリック
3. リージョンを選択する画面が出た場合は、東京リージョン（ap-northeast-1）を選択
   - 補足: IAM Identity Center自体は特定の1リージョンで有効化されますが、そこで管理する権限は全リージョンに対して使えます

## 4. 自分用のユーザーを作成する

1. IAM Identity Centerの画面で「ユーザー」→「ユーザーを追加」
2. ユーザー名、メールアドレス、氏名を入力して作成
   - 招待メールが届くので、そこからパスワード設定・MFA設定を行う

## 5. Permission Set（権限セット）を作成する

Permission Setは「どんな操作を許可するか」の定義です。学習目的なので、まずは広めの権限を割り当てます。

1. IAM Identity Centerの画面で「Permission sets」→「Permission setを作成」
2. 「事前定義のPermission set」を選択し、`AdministratorAccess` を選ぶ
   - これはAWSのほぼ全操作を許可する強い権限です。学習・個人利用なので今回はこれで進めますが、複数人で使う本番環境では絶対に避けるべき設定です
3. セット名はデフォルトのままでOK（例: `AdministratorAccess`）

## 6. ユーザーにアカウント・Permission Setを割り当てる

1. IAM Identity Centerの画面で「AWSアカウント」タブを開く
2. 対象のAWSアカウント（今作成したアカウント）にチェックを入れ、「ユーザーまたはグループを割り当てる」
3. 手順4で作成したユーザーを選択
4. 手順5で作成したPermission Set（`AdministratorAccess`）を選択して割り当て完了

## 7. アクセスポータルURLを確認する

1. IAM Identity Centerの「設定」ページを開く
2. 「AWSアクセスポータルのURL」をメモしておく（例: `https://d-xxxxxxxxxx.awsapps.com/start`）

このURLは次の `02-cli-setup.md` で `aws configure sso` を実行する際に必要になります。
